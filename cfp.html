What we're looking for

We want CampJS to emphasise hands-on activities and focussed talks on cutting-edge client and serverside technology.

  • ES6/7/8
  • Offline First
  • Web Components & other new/emerging web standards.
  • Browser Internals (e.g. rendering).
  • Node/Libuv/V8 Internals
  • Modular Application Design i.e. not frameworks.
  • Alternative Databases
  • Taking JavaScript to new environments (e.g. robots, native apps, operating systems)
  • Core Node (streams, tcp, etc) i.e. not frameworks.
  • JavaScript-powered Games (front/backend)
  • Original Modules or Research
  • Shell & editor tooling

Talks

We want to experiment with 'pair' presentations, i.e. every presentation is run by at least 2 people. This should remove some preparation burden, provide some level of sanity checking, and hopefully make things a bit fun. It is not required for both presenters to speak; Perhaps while one is speaking, the other is interpreting their words with dance (?). Get creative!

Talks should be limited to 20 minutes. If you can't fit your content in 20 minutes, perhaps you should consider running a workshop.

If these constraints are not suitable for something you'd like to present, please let us know. These are just guidelines, we are flexible!

Workshops

We want to run more, better workshops at CampJS. Workshops can run up to 90 minutes, but shorter is better. Workshops are limited to 20 people at a time, and should repeat at least once. Minimum 2 tutors. Ideally, workshops should be entirely self-directed, automated, and available online for anyone to participate. Tutors should only be needed to guide and answer questions.

We recommend use of the locally-grown workshopper framework. If workshopper doesn't do what you need, please consider submitting a pull request to improve it.
